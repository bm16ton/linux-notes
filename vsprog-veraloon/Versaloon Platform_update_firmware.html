Install Under Windows

There are 2 drivers for Versaloon, one is stmcdc.inf under dongle/driver/win32 directory, the other is libusb. If stmcdc.inf is used as driver for Versaloon, you can use Versaloon as a CDC tool. If you want to use vsprog/openocd/urjtag, you should use libusb as driver. Use inf-wizard.exe in the latest Libusb-WIN32 to generate this driver. Note that if you want to change the driver for Versaloon, you must remove original driver first and than install the new one.

Install Under Linux

Install libusb and libusb-dev. When Versaloon is connected, default cdc-acm is used as driver, and ttyACMx will appear under /dev directory. When running vsprog/openocd, Versaloon will be detached from the kernel driver(cdc-acm), and libusb will be used. Note that, default USB API layer in UrJTAG will not detach Versaloon from kernel driver, so you should run "vsprog -p versaloon" first.

Firmware Update

Run "vsprog -Z" after connect Versaloon, and after 3 seconds, Versaloon will reset to bootloader mode. Or simply push the button on Versaloon hardware, and connect it to PC, Versaloon will also run in bootloader mode. Use stmcdc.inf as driver under Windows and a COMx or ttyACMx will appear. Simply download application from 0x08002000 using any STM32_ISP tools.
